## 4. 开服前必须先确认的关键缺口

这一步非常重要，因为当前包不是完整的成品服。

### 4.1 服务端发布目录已就绪

`Server\运行.bat` 中启动命令是：

```bat
cd ./Bin
start "Realm 2" dotnet App.dll --AppId=2 --AppType=Realm --ConfigPath=../Config --LogLevel=Info
```

当前目录下已经有 `Server\Bin`，包含：

- `Server\Bin\App.dll`
- `Server\Bin\*.dll`
- `Server\Bin\Hotfix`
- `Server\Bin\temp` 等运行所需目录

服务端已经编译发布完成，可以直接使用 `Server\运行.bat` 启动。

### 4.2 StartUpConfig 启动配置已就绪

服务端源码里明确写死会读取：

- `ConfigPath\StartUpConfig\Server_DataConfig.json`
- `ConfigPath\StartUpConfig\StartUp_ServerConfig.json`

当前仓库里已经有 `Server\Config\StartUpConfig`，包含了这两个配置文件。如果需要修改区服配置，可以通过 Excel 表重新生成。

### 4.3 MongoDB 端口存在冲突

当前仓库中出现了两套不同端口：

- GM 工具配置使用 `27017`
- `DbTool` 源码硬编码连接 `58030`

具体如下：

- `Tools\webgm\config\db.js`
  使用 `mongodb://127.0.0.1:27017/...`
- `Server\Server\DbTool\Program.cs`
  使用 `mongodb://127.0.0.1:58030`

这说明你必须统一数据库端口，否则会出现：

- GM 能连上，DbTool 不能连
- DbTool 能建索引，GM 连不上
- 服务端区服配置指向的端口与实际 Mongo 不一致

建议你在正式部署前统一为一个端口，并同步修改所有相关配置。

## 17. 常见问题

### 17.1 双击 `Server\运行.bat` 没反应

常见原因：

- 没有 `Server\Bin\App.dll`
- 没有安装 .NET Runtime
- `Config\StartUpConfig` 缺失
- `Config\NewConfig` 不完整

### 17.2 GM 页面能打开，但操作报“GM 服务未开启”

先检查：

- `GM 99` 进程是否正常启动
- `Tools\webgm\config\db.js` 里的 `gmUrl` 是否正确
- 当前默认值是否仍为 `http://127.0.0.1:65001`

### 17.3 GM 能登录，但读不到玩家数据

通常是 Mongo 配置不一致：

- `db.js` 用的是 `27017`
- 服务端或 DbTool 用的是别的端口
- 分区库名和实际库名不一致

### 17.4 DbTool 建索引失败

大概率是：

- 端口 `58030` 不存在
- Mongo 没启动
- 你实际使用的库名不是 `ET / ET1 / ETLog`

### 17.5 文件服能启动，但客户端更新失败

重点检查：

- `Server\FileServer\appsettings.json` 的 `DirectoryPath`
- 资源目录里是否真的有发布文件
- 客户端里指向的资源 URL 是否与你的文件服地址一致
