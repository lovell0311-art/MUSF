# MUSF 依赖路径问题修复脚本

## 问题说明

**现象：**
- 登录页有背景，但输入框和按钮不可见
- 点击账号密码位置有反应（逻辑正常，UI资源缺失）

**根因：**
uilogin.unity3d.manifest 中的依赖是绝对路径，导致 Unity 无法正确打包依赖关系：
```
- F:/MUSF/Client/Unity/../../Release_Test/Android/StreamingAssets/ui_common_btnss.unity3d
- F:/MUSF/Client/Unity/../../Release_Test/Android/StreamingAssets/ui_common_inputfilledbgs.unity3d
```

---

## 脚本说明

### 1. build_fix_dependencies.bat
**完整的自动化构建脚本**
- 重新构建 AssetBundle（修复依赖路径）
- 重新构建 Code Bundle
- 打包 APK
- 卸载旧版本
- 安装新 APK
- 启动应用

**使用方法：**
```cmd
F:\MUSF\build_fix_dependencies.bat
```

### 2. check_apk_resources.ps1
**检查 APK 中的关键资源**
- 验证登录页相关资源是否存在
- 检查 manifest 格式
- 识别缺失资源

**使用方法：**
```powershell
powershell -ExecutionPolicy Bypass -File F:\MUSF\check_apk_resources.ps1
```

### 3. view_logs.ps1
**实时查看应用日志**
- 高亮显示关键日志（LoginStage、Bundle、UI等）
- 过滤无关信息

**使用方法：**
```powershell
powershell -ExecutionPolicy Bypass -File F:\MUSF\view_logs.ps1
```

或手动执行：
```cmd
C:\Android\Sdk\platform-tools\adb.exe -s 127.0.0.1:7555 logcat -v time | findstr "Unity uilogin ui_common login"
```

## 常见问题

### Q: 构建失败，提示 Unity 不存在
**A:** 修改脚本中的 Unity 路径：
```cmd
set UNITY_PATH=你的Unity安装路径\Editor\Unity.exe
```

### Q: APK 安装失败
**A:**
1. 检查 ADB 连接：`adb devices`
2. 手动卸载：`adb uninstall com.newquanzhilieshou.bytedance.gamecenter`
3. 重新安装

### Q: 资源仍然缺失
**A:**
1. 检查 Unity 编辑器日志
2. 确认 AssetBundle 构建没有报错
3. 删除 Assets/StreamingAssets 目录重新构建

### Q: 权限错误
**A:** 以管理员身份运行脚本
